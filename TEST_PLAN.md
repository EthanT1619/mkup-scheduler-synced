## 6. localStorage migration

| # | 시나리오 | 기대 결과 | 통과 |
|---|----------|-----------|------|
| 6.1 | localStorage 에 기존 일정 있음 + 최초 로그인 | 가져오기 안내 표시 | |

| 6.3 | 가져오기 후 localStorage 원본 | **자동 삭제되지 않음** | |

| 6.6 | 수동 localStorage 삭제 버튼 | 원본만 삭제, Supabase 유지 | |
| 6.7 | 이미 Supabase 에 있는 동일 일정 | 중복 삽입 건너뜀 (skipped) | |

## 테스트 순서 (권장)

1. 설정 오류 화면 (config 비움) → config 입력
2. 로그인 / 로그아웃 / 세션 유지
3. CRUD 기본 (추가·수정·삭제·상태)
4. 달력·필터·검색·타임라인·학생 회귀
5. PC ↔ 휴대폰 동기화
6. 계정 A / B 격리
7. localStorage migration (기존 데이터 있는 브라우저)
8. 네트워크 오류·다시 시도
9. `SECURITY_CHECKLIST.md` 항목 점검
